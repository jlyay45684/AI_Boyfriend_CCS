# CLAUDE STARTUP

Version: CCS v1.1

## Purpose

This document is the mandatory startup procedure for every Claude agent operating within the CCS repository.

Every Claude agent must execute this startup sequence before performing any work.

---

# STEP 1

Read and obey:

master/CLAUDE_CONSTITUTION.md

---

# STEP 2

Read:

master/CLAUDE_PLAYBOOK.md

---

# STEP 3

Read:

master/CLAUDE_ROLES.md

---

# STEP 4

Read current repository status:

orchestrator/SYSTEM_STATE.md

---

# STEP 5

Read:

handoff/

to understand the current project state.

---

# STEP 6

Read:

inbox/

to understand the latest completed work.

Never repeat completed phases.

Continue from the current repository state.

---

# STEP 7

Determine your assigned role.

Only execute responsibilities defined in:

master/CLAUDE_ROLES.md

Never perform another Claude agent's work.

---

# STEP 8

Read only the files required by your assigned role.

Avoid reading unrelated specifications unless necessary.

---

# STEP 9

Generate all outputs inside:

/inbox/

Never overwrite previous artifacts.

Always create a new version when revisions are needed.

Example:

claude_patch.md

claude_patch_v2.md

runtime_patch_v2.md

---

# STEP 10

Wait for human review.

Never continue automatically.

Repository owner approval is required before entering the next phase.

---

End of Startup
